- [ ] Три вкладки: «Аудит» · «AI-аудит» · «Настройки» (без «Правила» и «Игноры»)

- [ ] Повторный запуск плагина читает каталог из кеша (<200 мс до отрисовки)

## Вкладка «Аудит» (автоаудит)

### Empty states
- [ ] Без выделения: empty-state «Выделите фрейм» с иконкой MousePointer, CTA-кнопки нет
- [ ] С выделением, до прогона: «Запустите аудит макета» + описание + иконка Play (без дублирующей кнопки, primary CTA в футере)
- [ ] После прогона без проблем: «Отличный макет» с CheckCircle2 (зелёный success tone)
- [ ] С активным фильтром, который отсеял всё: «Под фильтры ничего нет» + CTA «Сбросить фильтры»

### Шапка (score + filters)
- [ ] Score 0–100 с вердиктом-бейджем (Критично/Плохо/Средне/Хорошо/Отлично) соответствующего цвета
- [ ] Progress-bar заполнен на `score%`, цвет соответствует вердикту
- [ ] Строка справа: «N проблем · M правил» с русской плюрализацией (1 проблема / 2 проблемы / 5 проблем)
- [ ] Severity-чипсы `228 ошибок`, `99 предупреждений`, `18 советов` — кликабельные, active state = сплошной цвет severity
- [ ] Селект фильтра по правилу показывает только правила, встречающиеся в текущих issues
- [ ] При активном rule-фильтре справа появляется кнопка `✕` сброса
- [ ] Под фильтрами появляется строка «Показано X из Y» когда фильтр активен

### Карточки issue
- [ ] Свёрнутая карточка: severity-иконка, title, P-бейдж, кнопка `→` справа (jump)
- [ ] Клик на карточку раскрывает детали: Что / Где / Почему / Как исправить + действия
- [ ] Кнопка «Перейти» точно уводит на конкретный слой (не на корневой фрейм) — работает даже если слой на другой странице
- [ ] Кнопка «Скопировать fix» кладёт markdown-сниппет в буфер
- [ ] Кнопка «Документация →» открывает ссылку в новой вкладке если `docUrl` есть
- [ ] Одинаковые правила на N слоях схлопываются в одну карточку с бейджем `× N`, раскрываются в список слоёв с индивидуальными кнопками `→`

### Футер
- [ ] Кнопка «Запустить аудит» (primary, black) с иконкой Play
- [ ] Кнопка «Отчёт» (outline) с иконкой Download — копирует Markdown-отчёт в буфер
- [ ] Кнопка «Отчёт» disabled когда issues=0

## Вкладка «AI-аудит»

### Без API-ключа
- [ ] Empty-state с иконкой Key, текст про необходимость ключа, кнопка «Открыть настройки →»
- [ ] Клик на кнопку переключает на вкладку «Настройки»

### С валидным ключом
- [ ] Селектор модели в Настройках содержит: Haiku 4.5 / Sonnet 4.5 / Opus 4.5 / **Opus 4.7 (рекомендуется)**
- [ ] При прогоне: нативный тост Figma «[модель] · батч 1/3», «батч 2/3», «батч 3/3»
- [ ] 3 параллельных API-вызова; один упавший не валит весь прогон
- [ ] Карточки AI-issues не имеют метки «auto» — все в моно-режиме
- [ ] Кнопка «Перейти» в AI-карточке работает благодаря fuzzy-матчу `nodeHint` против имён слоёв / текста
- [ ] При 0 найденных проблем: empty-state «AI ничего не нашёл»

### Ошибки
- [ ] При HTTP 401/invalid key — тост «API-ключ не принят Anthropic», автоматический переход на вкладку Настройки
- [ ] При отсутствии сети — тост с ошибкой fetch, плагин не падает
- [ ] При фрейме >1 МБ после PNG-экспорта scale 2 — тост «Уменьшите масштаб»
- [ ] API-ключ с невидимыми юникод-символами (NBSP, фигурные кавычки) — чистится автоматически

## Формула score

Проверить, что вердикт соответствует вычислению `100 / (1 + penalty/50)` где `penalty = errors × 5 + warnings × 2 + infos × 0.5`:

- 0/0/0 → 100 (Отлично)
- 1/0/0 → 91 (Отлично)
- 3/5/2 → 72 (Хорошо)
- 10/10/10 → 37 (Плохо)
- 77/38/18 → 10 (Критично)
