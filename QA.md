# QA-чеклист — Design Libs Inspector

Выполнять перед сабмитом в Figma Community. Версия: см. `package.json`.

## Подготовка

- [ ] `npm run typecheck` → 0 ошибок
- [ ] `npm test` → все тесты зелёные, бенчмарк p95 <500 мс
- [ ] `npm run build` → `dist/code.js` + `dist/ui.html`, общий размер <5 МБ
- [ ] API `GET https://design-libs.vercel.app/api/figma-rules/v1` отдаёт 200 и count ≥ 50
- [ ] В Figma desktop: `Plugins → Development → Import plugin from manifest…` → выбрать `manifest.json`

## Базовый прогон

- [ ] Плагин открывается без JS-ошибок в консоли Figma (`Plugins → Development → Open Console`)
- [ ] Header показывает имя текущего документа и вкладки «Проблемы / Правила / Игноры / Настройки»
- [ ] При старте каталог правил загружается за <2 с (проверить счётчик «Правил в кеше» в «Настройках»)
- [ ] Повторный запуск плагина читает каталог из кеша (<200 мс до отрисовки вкладки)

## Сценарий «Проблемы»

- [ ] Без выделения: пустое состояние «Выделите фрейм для аудита», кнопка «Запустить аудит» disabled
- [ ] С выделенным фреймом: кнопка «Запустить аудит» активна, после клика появляется список проблем
- [ ] Оценка 0–100 соответствует формуле (ошибка −10, предупреждение −3, совет −1)
- [ ] Чипы severity фильтруют список; повторный клик по чипу снимает фильтр
- [ ] Клик по имени ноды в карточке проблемы — выделяет и центрирует слой в Figma (`viewport.scrollAndZoomIntoView`)
- [ ] Кнопка «Игнорировать» убирает issue и добавляет запись на вкладке «Игноры»
- [ ] Кнопка «MD» копирует отчёт в буфер обмена; вставить в редактор — markdown валиден

## Сценарий AI

- [ ] Без API-ключа кнопка «AI-анализ» disabled, в toast — «Введите API-ключ в «Настройках»»
- [ ] С валидным `sk-ant-…` ключом (Haiku для теста) прогон выдаёт результат за 3–8 с
- [ ] Карточки AI-issues помечены бейджем «AI» и имеют валидный ruleId из каталога
- [ ] При HTTP 401 (неверный ключ) — toast с сообщением из ответа Anthropic
- [ ] При отсутствии сети — toast с ошибкой fetch, плагин не падает
- [ ] При выделении фрейма >1 МБ при scale 2 — toast «Уменьшите масштаб экспорта»

## Вкладка «Правила»

- [ ] Без выделения — пустое состояние с подсказкой
- [ ] С выделенной кнопкой — показываются правила типа `button | focus | disabled | contrast`
- [ ] С выделенным текстом — правила типа `typography | font | text | contrast`
- [ ] Кнопка «Все правила · 227» показывает полный каталог
- [ ] Категория и anti-pattern отрисованы

## Вкладка «Игноры»

- [ ] Список игноров сохраняется после `Plugins → Close plugin` + повторный запуск
- [ ] Игноры видны другому пользователю, открывшему тот же .fig-файл (через `pluginData`)
- [ ] Кнопка «Вернуть» возвращает проблему в прогоне «Проблем»
- [ ] Счётчик в заголовке совпадает с длиной списка

## Настройки

- [ ] API-ключ вводится в password-поле, не логируется, сохраняется в `clientStorage` (per-user)
- [ ] Выбор модели Haiku / Sonnet / Opus сохраняется между сессиями
- [ ] Платформа (web / ios / android) влияет на порог touch-target в следующем прогоне
- [ ] Сетка 4/8 px влияет на grid-snap
- [ ] Чекбокс «Тёмная тема» включает `dark-theme-purity` и меняет дефолтный bg для контраста
- [ ] «Обновить каталог» тянет свежий JSON и обновляет счётчик
- [ ] «Очистить кеш» обнуляет список правил

## Крайние случаи

- [ ] Плагин не ломается на пустом фрейме (0 детей)
- [ ] Корректно работает на большом фрейме (≥500 слоёв) — сериализатор обрезает до 2000 нод, UI не фризит
- [ ] Имена слоёв с кавычками/эмодзи отображаются в карточках без разломов верстки
- [ ] Смена темы Figma (светлая ↔ тёмная) применяется к UI через `--figma-color-*` без перезапуска
