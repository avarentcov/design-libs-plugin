# Community assets

Ассеты для публикации в Figma Community.

| Файл | Требование Figma |
|---|---|
| `icon.svg` | 128×128, SVG/PNG |
| `cover.svg` | 1920×1080 (экспортировать в PNG перед загрузкой) |
| `screenshots/*` | 6 изображений 1920×1080 PNG (добавить перед сабмитом) |
| `description.md` | ≤1000 символов RU |
| `tags.json` | 12 тегов |

## Экспорт SVG → PNG перед загрузкой

```bash
# требует установленный rsvg-convert или inkscape
rsvg-convert -w 128 -h 128 icon.svg -o icon.png
rsvg-convert -w 1920 -h 1080 cover.svg -o cover.png
```

## Чек-лист скриншотов (6 штук, 1920×1080)

1. **Вкладка «Аудит»** — найдено ~10 проблем, видны score 0–100 с цветным вердиктом, progress-bar, severity-чипсы с числами (228 ошибок · 99 предупреждений · 18 советов), селект фильтра по правилу
2. **Раскрытая карточка** — секции Что / Где / Почему / Как исправить + действия (Перейти / Скопировать fix / Документация)
3. **Группированная карточка** с бейджем `× N` — одно правило на нескольких слоях, раскрыт список с индивидуальными кнопками «Перейти»
4. **Вкладка «AI-аудит»** — прогон Opus 4.7 с deep-thinking, 20+ issues по композиции/иерархии/формам
5. **Вкладка «Настройки»** — поле API-ключа с маской и зелёным индикатором, селектор модели (Opus 4.7 рекомендуется), блок с версией каталога и price note
6. **Empty-states подряд** (для cover-варианта) — «Выделите фрейм» / «Запустите аудит» / «Отличный макет» с иконками

Сделайте на реальном фрейме после финального билда.
